# Technical Writer in Verify

You are here to make sure what ships has docs people can actually use. Verify is where you walk the product with the docs — step by step, claim by claim — alongside QA/Tester to catch drift, missing pages, and stale screenshots before users hit them.

## What you do

- Review documentation against the shipping product jointly with QA/Tester — the docs walk-through.
- Catch terminology, accuracy, and coverage gaps and close them fast.
- Align with UI/UX Designer on in-product copy as visual QA surfaces mismatches.
- Prepare final feature docs and user guides for Launch.

## Artifacts you own

- [verified docs](artifacts/verified-docs.md) — co-owned with QA/Tester.

## Artifacts you contribute to

You don't formally contribute to other Verify-phase artifacts beyond the docs review.

## Outcomes you drive

You don't drive outcomes this phase — you input into others'.

## Who you work with

- **QA/Tester** — co-owns documentation review; primary counterpart for the walk-through.
- **UI/UX Designer** — aligns on in-product copy.
- **Developer** — fixes inaccurate or missing API docs you flag.
- **Customer Support** — confirms docs meet their ticket-deflection needs.

## Handoff into [Launch](../7-launch/README.md)

In Launch you publish feature documentation and user guides. You know Verify is done when the walk-through finds no blockers, the docs accurately describe the shipping product, and your Launch-phase drafts are ready to publish.
