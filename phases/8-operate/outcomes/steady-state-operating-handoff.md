# steady-state operating handoff

_Produced by: Hypercare transition to steady-state_

**Primary owner:** Project Manager

## What this is

The PM formally transitions the initiative from project-mode (with governance, RAID log, change control) to steady-state operations.

## Why it matters

Projects end; products continue. Without a handoff, project-era governance either lingers as ceremony or evaporates without a replacement.

## What it contains

- Steady-state owners named
- Operating cadence established
- Project-era artifacts archived

## Definition of Done

- [ ] Steady-state owners are named for each operational area.
- [ ] The operating cadence (meetings, reports, escalations) is documented and running.
- [ ] Project-era artifacts (RAID log, change control, governance) are archived, not active.
- [ ] The PM has a recorded sign-off closing the project.

## Entry criteria

- Launch stabilized
- Operating cadence running

## Exit signal

The PM can close the project and the product runs on operational rhythms.
