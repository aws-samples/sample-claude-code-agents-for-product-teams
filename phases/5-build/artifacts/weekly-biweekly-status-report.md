# weekly/biweekly status report

_Produced by: Sponsor/PO status report preparation_

**Business outcome supported:** produce working, tested, instrumented increments that meet the Definition of Done, with stakeholder-accepted quality and documented lessons for continuous improvement.

**Primary owner:** Project Manager

**Stakeholders:** _(none listed)_

**Accelerated MVP:** optional when status lives on the board and in demos

## What this is

The PM's recurring delivery status — progress against baseline, feature/epic state, top risks and issues, scope changes, budget burn, and help-needed. Written for the Sponsor and PO; detailed enough that the exec status update can be derived from it.

## Why it matters for the Project Manager

Regular status is how you convert the complexity of delivery into a signal stakeholders can act on. A report that is honest, specific, and on-cadence builds the trust that lets you protect the team when things get difficult.

## Definition of Done

- [ ] Overall RAG includes a one-line rationale
- [ ] Progress is shown against baseline milestones
- [ ] Top 3-5 risks and issues list mitigation state
- [ ] Budget and burn position is stated
- [ ] Every help-needed ask names a decision owner

## What it contains

- Overall RAG with one-line rationale
- Progress against baseline milestones
- Feature/epic state summary
- Top 3–5 risks and issues with mitigation state
- Scope changes in-flight
- Budget and burn position
- Help-needed asks with named decision owners

## Inputs you rely on

- Feature/epic status report
- RAID log and risk register
- Budget burn report
- Architecture status report and security/compliance inputs
- Iteration plan and assigned workitems

## Who consumes it

- Product Sponsor — derives executive status update
- Product Owner — sees delivery health against backlog sequence
- Governance forums — PMO, steering
- Adjacent PMs — coordinate cross-initiative touchpoints

## Common pitfalls

- Green across the board while blockers fester — nobody trusts the report
- Narrative buried — executives can't extract the signal in 60 seconds
- Asks left implicit — help-needed becomes help-missed
- Report becomes copy-paste from last week with state changes lost
