# Technical Writer in Plan

Plan is a quiet phase for you — documentation production ramps in Verify and peaks at Launch. Your job here is to get the docs-as-code toolchain ready, understand the product scope so you can plan authoring, and make sure API/inline docs are part of the Definition of Done developers are signing up for.

## What you do

- Stand up or refresh the docs-as-code toolchain so it's ready when content starts landing.
- Review the decomposed backlog to understand what will need user-facing documentation at Launch.
- Align with Developers on the API/inline doc generation pattern — what they write, what gets auto-published.
- Begin outlining the feature documentation and user guide structure for Launch.

## Artifacts you own

You don't own any artifacts this phase — you contribute to others'.

## Artifacts you contribute to

You don't formally contribute to a Plan-phase artifact — your contribution is toolchain readiness and authoring prep.

## Outcomes you drive

You don't drive outcomes this phase — you input into others'.

## Who you work with

- **Developer** — align on API doc generation and inline documentation expectations.
- **Product Owner** — review scope so you can plan docs effort against the roadmap.
- **UI/UX Designer** — align on tone, terminology, and any in-product copy you'll own.

## Handoff into [Build](../5-build/README.md)

In Build you'll start producing drafts alongside feature code — in-product copy, API reference, and first-pass user guides. In Verify you'll review docs against the shipping product. You know Plan is done when the docs pipeline builds, you have a content outline tied to the backlog, and developers know what inline/API docs are expected.
