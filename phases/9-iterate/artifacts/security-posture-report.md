# security posture report

_Produced by: Security posture review_

**Business outcome supported:** close the loop — measure actual performance against the business case, decide continue/pivot/sunset, and feed validated learning back into Discover for the next cycle.

**Primary owner:** Security & Compliance

**Stakeholders:** _(none listed)_

**Accelerated MVP:** slim — the MVP's threat-model check-in is enough

## What this is

The periodic assessment of the product's overall security posture — control maturity, residual risk, incident trend, audit standing, and recommended investments. Written for the Sponsor and executive leadership, not just the security team.

## Why it matters for the Security & Compliance team

Operational security data is everywhere (SIEM, scan reports, patch logs); executives need it distilled into a decision-ready view of risk and maturity. This report is how security earns continued investment without having to explain every detection rule to a CFO.

## Definition of Done

- [ ] Control maturity is scored against the chosen framework
- [ ] Residual risk heatmap with top risks is included
- [ ] Incident and vulnerability trend is reported
- [ ] Audit findings and remediation status are listed
- [ ] Investment asks are stated with business impact framing

## What it contains

- Control maturity against chosen framework
- Residual risk heatmap with top risks
- Incident and vulnerability trend
- Audit findings and remediation status
- Recurring vuln classes and systemic fixes
- Investment asks with business impact

## Inputs you rely on

- Audit evidence and audit report / attestation
- Security monitoring alerts and security incident postmortems
- Patched dependencies and access review results
- Threat model (updated)
- Regulatory change impact

## Who consumes it

- Sponsor and CISO — investment and risk-acceptance decisions
- Architect — feeds architecture roadmap and modernization plan
- Sales — enterprise deal security responses
- Next Discover cycle — security considerations for new bets

## Common pitfalls

- Jargon that executives tune out; no translation to business risk
- Green-washing the heatmap
- Recurring findings unaddressed; report identical quarter to quarter
- Asks without ROI framing; security investment gets deprioritized
