# build-vs-buy decision

_Produced by: Approve build-vs-buy & vendor decisions_

**Primary owner:** Product Sponsor

**Stakeholders:** Architect

**Accelerated MVP:** required — the decision is real even when the write-up is slim

## What this is

The decision to build a capability in-house, buy it from a vendor, partner for it, or use an existing enterprise capability — with the rationale recorded.

## Why it matters

This decision sets cost structure, time-to-market, and strategic positioning. It must be made explicitly and revisited only with discipline.

## What it contains

- Decision for each material capability: build / buy / partner / reuse
- Rationale, including TCO, strategic control, and speed
- Lock-in and exit considerations for any 'buy' choice

## Definition of Done

- [ ] A build / buy / partner / reuse decision is recorded for every material capability.
- [ ] Rationale cites TCO, strategic control, and speed-to-market.
- [ ] Any "buy" choice names the lock-in and exit considerations.
- [ ] Sponsor ratification is recorded alongside the decision.

## Entry criteria

- Vendor evaluation scorecard complete
- Vendor security & compliance assessment complete
- Refined ROI model includes both build and buy scenarios

## Exit signal

Vendor contracting or internal build can begin without further debate about direction.
