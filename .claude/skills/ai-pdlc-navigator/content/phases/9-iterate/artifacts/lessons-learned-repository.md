# lessons-learned repository

_Produced by: Lessons-learned facilitation_

**Business outcome supported:** close the loop — measure actual performance against the business case, decide continue/pivot/sunset, and feed validated learning back into Discover for the next cycle.

**Primary owner:** Project Manager

**Stakeholders:** _(none listed)_

## What this is

The organization-wide, searchable repository of lessons captured from this initiative — what worked, what didn't, what surprised the team — tagged by phase, role, and theme so future projects can pull from it.

## Why it matters for the Project Manager

Retrospectives that live only in one team's memory compound nothing. A shared repository is how the organization improves PDLC maturity over time instead of relearning the same costly mistakes. It is one of the most leveraged artifacts the PMO owns.

## Definition of Done

- [ ] Each lesson is tagged by phase, role, and topic
- [ ] Context (what was attempted, what happened) is captured per lesson
- [ ] A recommendation for future projects is stated
- [ ] Status (adopted into standards, still open) is tracked
- [ ] Source retrospective or RCA is linked for each entry

## What it contains

- Lessons tagged by phase, role, and topic
- Context (what was being attempted, what happened)
- Recommendation for future projects
- Status (adopted into standards, still open)
- Links to retrospectives and RCAs that sourced them
- Search-friendly taxonomy

## Inputs you rely on

- Retrospective report from this project
- RCAs and incident postmortems
- Process improvement actions from Build
- Stakeholder feedback from closure interviews
- Prior-project lessons that applied (or didn't)

## Who consumes it

- Future project teams — check before key decisions
- PMO and engineering leadership — feed into standards and templates
- Onboarding — reading list for new leads
- Next Discover cycle — calibrate feasibility claims

## Common pitfalls

- Dump-and-forget — no curation, no one searches it
- Vague lessons ("communicate better") that don't change behavior
- Lessons captured but not reflected in updated templates or playbooks
- Access restricted; the org never benefits
