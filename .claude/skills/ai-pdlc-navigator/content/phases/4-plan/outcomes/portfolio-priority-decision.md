# portfolio priority decision

_Produced by: Resolve cross-portfolio priority conflicts_

**Primary owner:** Product Sponsor

**Stakeholders:** Architect

## What this is

The Sponsor resolves conflicts where this initiative and other portfolio work compete for the same scarce resources (shared platforms, senior talent, regulatory bandwidth).

## Why it matters

Portfolio conflict is invisible to the individual project until it blocks. The Sponsor is the only person with the standing to arbitrate.

## What it contains

- Specific conflicts and the chosen resolution
- Impact to affected initiatives
- Re-visit triggers if circumstances change

## Definition of Done

- [ ] Each conflict is named with the chosen resolution
- [ ] Impact to each affected initiative is documented
- [ ] Re-visit triggers are specified in case circumstances change
- [ ] Sponsor decision is recorded with date

## Entry criteria

- Dependencies and shared-platform needs identified
- Architect has flagged cross-portfolio conflicts

## Exit signal

The project plan is consistent with the rest of the portfolio, not competing with it.
