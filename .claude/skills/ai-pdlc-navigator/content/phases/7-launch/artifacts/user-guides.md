# user guides

_Produced by: Draft user guides_

**Business outcome supported:** product live, teams enabled, customers notified, baseline metrics captured, and launch stabilized through hypercare.

**Primary owner:** Technical Writer

**Stakeholders:** _(none listed)_

## What this is

Task-oriented, narrative guides that walk a user from zero to outcome — getting started, common workflows, troubleshooting. Complements feature documentation (which is reference) with flow-driven content.

## Why it matters for the Technical Writer

Activation and time-to-first-value depend on this content. A good user guide deflects setup tickets, accelerates trials, and becomes a sales asset; a weak one forces support to hand-hold every new customer through onboarding.

## Definition of Done

- [ ] Getting started / quickstart path is published and linked from the product entry point.
- [ ] End-to-end workflow guides exist for the top 3–7 jobs-to-be-done.
- [ ] Troubleshooting decision trees cover the common failure modes.
- [ ] Glossary of product-specific terms is included.
- [ ] Headings and anchor links are structured for search discoverability.

## What it contains

- Getting started / quickstart path
- End-to-end workflows for the top 3–7 jobs-to-be-done
- Troubleshooting decision trees for common failure modes
- Glossary of product-specific terms
- Screenshots, diagrams, and short videos where they help
- Search-friendly headings and anchor links

## Inputs you rely on

- User personas and journey maps from Discover and Design
- Feature documentation for link-throughs
- Design partner and UAT feedback on pain points
- Known-issues list and workarounds
- Content style guide and design-system copy standards

## Who consumes it

- New users and trial evaluators — self-serve onboarding
- Customer Support — link resolutions instead of writing tickets
- Customer Success — shape onboarding playbooks and QBRs
- Sales Engineering — hand to prospects in evaluation

## Common pitfalls

- Writing for the product team's mental model instead of the user's
- One giant document instead of tasks the user can search to
- Out-of-date screenshots and flows after the next release
- Hiding troubleshooting deep; users never find it when they need it
