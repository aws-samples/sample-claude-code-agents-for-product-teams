# Technical Writer in Iterate

Iterate is a reflection phase for you. You're not the lead on any Iterate-phase artifact, but a cycle of Operate-phase KB analytics, ticket tags, and search data is the best signal you'll get for where documentation earned its keep and where it didn't. Your job is to translate that into next-cycle docs investment.

## What you do

- Synthesize KB analytics and support-ticket tagging into next-cycle docs recommendations.
- Identify documentation gaps that persisted through Operate — these are product or docs signal.
- Contribute to the lessons-learned repository on docs toolchain and cross-team contribution patterns.
- Feed docs-driven opportunity signal into the next-cycle backlog.

## Artifacts you own

You don't own any artifacts this phase — you contribute to others'.

## Artifacts you contribute to

You don't contribute to others' artifacts this phase.

## Outcomes you drive

You don't drive outcomes this phase — you input into others'.

## Who you work with

- **Customer Support** — bi-directional on ticket trends that expose docs gaps.
- **Product Owner** — bi-directional on docs gaps that belong in the next-cycle opportunity backlog.
- **Project Manager** — bi-directional on lessons-learned capture for docs-as-code and contribution flow.
- **UI/UX Designer** — bi-directional on journey-layer doc patterns.

## Handoff into [the next Discover cycle](../1-discover/README.md)

You exit Iterate with a cycle of real KB usage evidence. Carry that into the next Discover cycle — documentation needs identified at Discover or Define cost a fraction of what they cost at Launch, so the patterns you surface here help next cycle's docs investment land in the right phase.
