# Release Manager in Iterate

Iterate is a reflection phase for you. You're not the lead on any Iterate-phase artifact, but your Operate-phase release data — change-failure rate, rollback exercises, hypercare duration — is what the retrospective needs to improve release integrity on the next cycle.

## What you do

- Synthesize Operate-phase release data into next-cycle release-integrity recommendations.
- Contribute change-failure and MTTR patterns to the post-launch retrospective.
- Keep the release playbook and rollback discipline updated with cycle learnings.
- Identify the release-engineering investments the next cycle needs.

## Artifacts you own

You don't own any artifacts this phase — you contribute to others'.

## Artifacts you contribute to

You don't contribute to others' artifacts this phase.

## Outcomes you drive

You don't drive outcomes this phase — you input into others'.

## Who you work with

- **Project Manager** — bi-directional on the retrospective and lessons-learned capture.
- **Site Reliability Engineer** — bi-directional on release-reliability lessons.
- **Developer** — bi-directional on release-quality lessons.
- **QA/Tester** — bi-directional on release-verification lessons.

## Handoff into [the next Discover cycle](../1-discover/README.md)

You exit Iterate with real release-integrity evidence. Carry that into the next Discover cycle's feasibility memo and initial project brief — next-cycle release planning gets sharper every time you hand forward what this cycle taught you about deployability, rollback, and cadence.
