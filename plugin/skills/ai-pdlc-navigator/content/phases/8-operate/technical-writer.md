# Technical Writer in Operate

Operate is where docs either deflect support load or generate it. You curate the customer-facing knowledge base, using ticket signal and search data to figure out where documentation is earning its keep and where it's failing. Your job is measurable time-to-value and lower cost-to-serve on documented topics.

## What you do

- Curate the customer-facing knowledge base — what's authoritative, what's deprecated, what's missing.
- Use support ticket tagging and search analytics to find doc gaps and update them.
- Patch docs on incident learnings so the "what we know about X" surface stays current.
- Keep docs-as-code infra healthy so contributors across roles can land changes cleanly.

## Artifacts you own

- [customer-facing knowledge base](artifacts/customer-facing-knowledge-base.md) — the curated, maintained KB that customers and Support both rely on.

## Artifacts you contribute to

You don't contribute to others' artifacts this phase.

## Outcomes you drive

You don't drive outcomes this phase — you input into others'.

## Who you work with

- **Customer Support** — bi-directional on ticket-tag data and doc gap identification.
- **Developer** — bi-directional on API reference accuracy and behavioral updates.
- **UI/UX Designer** — bi-directional on journey gaps surfaced by docs analytics.
- **Sales & Marketing** — bi-directional on messaging consistency.

## Handoff into [Iterate](../9-iterate/README.md)

You exit Operate with KB analytics showing where docs are and aren't working. Carry that signal into Iterate — docs gaps that persist through Operate are product signal, not just docs signal, and belong in the next-cycle opportunity backlog alongside bug and tech-debt work.
