# confirmed data classification

_Produced by: Data classification confirmation_

**Primary owner:** Security & Compliance

## What this is

Security & Compliance has confirmed the sensitivity tier of every data element the product handles — personal, financial, health, confidential, public.

## Why it matters

Classification drives which security controls apply, what regions can host data, which audits pay attention, and how incidents must be reported. Getting this wrong at Define means expensive rework at Verify.

## What it contains

- Data inventory with classification tier per element
- Regulatory triggers per tier (GDPR, HIPAA, PCI, etc.)
- Handling, retention, and residency requirements

## Definition of Done

- [ ] Every element in the data inventory has a confirmed classification tier
- [ ] Regulatory triggers are named per tier (GDPR / HIPAA / PCI / etc.)
- [ ] Handling, retention, and residency requirements are specified per tier
- [ ] Security & Compliance sign-off is recorded against the confirmed classification

## Entry criteria

- BA has drafted data requirements & classification
- Applicable-regulations list is current
- Privacy / DPIA assessment is in flight or complete

## Exit signal

Architect can design storage, access, and transfer paths that satisfy the classification.
