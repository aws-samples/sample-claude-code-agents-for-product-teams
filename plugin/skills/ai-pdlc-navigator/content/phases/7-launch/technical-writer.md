# Technical Writer in Launch

Launch is where time-to-value either happens or doesn't. You ship the feature documentation and user guides customers hit within minutes of landing on the product. Clean docs at GA deflect support tickets, enable sales, and set the bar for the knowledge base you'll curate in Operate.

## What you do

- Create feature documentation that matches what actually shipped, not the design spec.
- Draft user guides that walk real users through primary jobs-to-be-done.
- Coordinate with Sales & Marketing so messaging, UI copy, and docs are consistent.
- Capture doc gaps surfaced during hypercare for rapid patching.

## Artifacts you own

- [feature documentation](artifacts/feature-documentation.md) — the authoritative reference for what each feature does.
- [user guides](artifacts/user-guides.md) — the task-oriented walkthroughs that drive activation.

## Artifacts you contribute to

You don't contribute to others' artifacts this phase.

## Outcomes you drive

You don't drive outcomes this phase — you input into others'.

## Who you work with

- **Developer** — bi-directional on API reference accuracy and behavioral edge cases.
- **UI/UX Designer** — bi-directional on screenshot fidelity and flow alignment.
- **Sales & Marketing** — bi-directional on launch collateral consistency.
- **Customer Support** — bi-directional on doc gaps surfaced by early tickets.

## Handoff into [Operate](../8-operate/README.md)

You exit Launch with docs at GA and a first read on what real users look up. Carry that into Operate's customer-facing knowledge base curation — Support ticket tagging and search data will tell you exactly where your docs are strong, where they're missing, and where they're outright wrong. That's the signal for Operate-phase docs investment.
